# Cognitive Design Principles

Use this file during **Stage 3** and **Stage 8**.

Its purpose is to stop the deck from becoming a dense but pretty wall of content. Good slide design is a cognitive contract, not just a visual one.

## 1. Working memory budget

Use Cowan-style working-memory constraints rather than the older loose “7 plus or minus 2” habit.

- target **4 ± 1 information chunks** in active focus
- treat one coherent card as one chunk only if its contents read as a single unit
- if a slide needs more than 5 meaningful chunks, split it

## 2. Mayer multimedia rules

Apply these high-impact rules:

- **Spatial contiguity**: labels stay near the thing they explain
- **Coherence**: remove decorative noise, redundant text, and low-value visuals
- **Signaling**: use size, contrast, weight, and spacing to guide attention
- **Segmenting**: split complex explanations across slides rather than shrinking text
- **Redundancy**: do not dump full narration paragraphs onto the slide

## 3. Gestalt rules for slide layout

- **Proximity**: related elements sit together; unrelated groups get visible whitespace
- **Similarity**: same-level elements should share shape, tone, and sizing logic
- **Continuation**: align edges and reading paths so the eye does not zig-zag randomly
- **Figure-ground**: primary content must remain legible against the background
- **Common region**: cards and containers should clarify grouping, not add clutter

## 4. Three-tier hierarchy

Never use more than 3 hierarchy tiers on one slide.

- **Primary**: title, hero metric, or core claim
- **Secondary**: supporting argument, chart, or explanation
- **Tertiary**: citation, label, footnote, or annotation

If two things both look primary, the hierarchy is broken.

## 5. Three-second test

For each slide, ask:

1. What does the audience notice first in 3 seconds?
2. What do they notice second?
3. What is the single sentence they should remember?

If the answers are unclear, reduce density or rebuild hierarchy.

## 6. Slide-role guidance

Use `story_role` to control density:

- `anchor`: one strong claim, one supporting proof cluster
- `proof`: evidence-heavy but still chunked and cited
- `bridge`: transition or reframing slide; keep it light
- `breathing`: visual or summary reset; minimal content
- `closing`: recap plus next action

Bridge and breathing slides should not look like proof slides.

## 7. Density limits

- cover / breathing / closing: 1-3 information units
- comparison / process / KPI: usually 3-5 units
- dense proof slides: 4-6 units only if grouping is strong and citations are visible

When in doubt, split the slide.

## 8. Review questions

Use these during review:

- Is the key message obvious without reading every word?
- Does each card earn its existence?
- Is any paragraph doing speaker-note work instead of slide work?
- Did the design lower extraneous cognitive load or add to it?
