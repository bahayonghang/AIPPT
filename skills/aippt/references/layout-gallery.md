# Layout Gallery

Use this file in **Stage 4** and **Stage 5**.

This is the high-level layout vocabulary layer above `bento-grid-system.md`.

## Layout hints

| Layout hint | Best for | Usually maps to |
| --- | --- | --- |
| `title-hero` | opening claim, cover, section reset | `cover`, `single-focus` |
| `split-screen` | compare text and proof side by side | `asymmetric-two-column`, `symmetric-two-column`, `media-text` |
| `three-columns` | 3 parallel factors or options | `three-column` |
| `dashboard` | KPI stack, metrics, multi-card evidence | `two-by-two-dashboard`, `mixed-grid` |
| `timeline` | ordered stages or date sequence | `hero-plus-three`, `hero-plus-four`, `single-focus` |
| `hub-spoke` | ecosystem, dependency, org or architecture | `mixed-grid`, `hero-plus-three` |
| `bento-summary` | mixed overview with one focal block | `mixed-grid` |
| `case-strip` | example plus supporting proof rail | `media-text`, `hero-plus-two` |
| `quote-callout` | testimonial, user voice, thesis statement | `single-focus`, `hero-plus-two` |
| `decision-matrix` | options vs criteria | `symmetric-two-column`, `mixed-grid` |

## Selection rule

Choose `layout_hint` first for human readability, then choose the canonical `final_layout` that best executes it.

## Pacing rule

Do not repeat the same `layout_hint` three pages in a row unless the deck is a uniform appendix.
